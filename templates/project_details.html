Smart Safe Project Details

Smart Safe is an IoT-based face recognition security system designed to protect valuables by allowing access only to the registered owner.

1. Project Overview

This project combines IoT, face recognition, Python, Flask, camera module, LCD display, buzzer, and lock control. The main purpose is to create a smart safe that opens only when the authorized owner’s face is detected. It also ensures that no static images or pre-made videos are used by enabling liveliness using random expression verification, making it unpredicatble.

2. What is IoT?

IoT means Internet of Things. It is a technology where physical devices such as cameras, sensors, buzzers, motors, displays, and controllers communicate with software to perform automatic actions.

In this project, IoT is used to connect the software face recognition system with hardware components like buzzer, LCD, and safe lock.

3. What is Face Recognition?

Face recognition is a biometric technology that identifies a person using facial features. The camera captures the face, the software analyzes it, and then compares it with the registered owner’s face data.

4. Main Aim

The aim of this project is to build a keyless smart safe system that uses face recognition instead of keys or passwords. It increases security and reduces unauthorized access.

7. How the System Works

  1. The owner registers their face.
  2. The face data is stored in the system.
  3. The user clicks recognition.
  4. The camera scans the face.
  5. The software compares the scanned face with stored owner data.
  6. If matched, it goes ahead with the Expression Verification.
  7. If not matched, access is denied.
  8. Random expression is generated.
  9. If the expression is performed accurately, access is granted.
  10. If not matched, access is denied.

WHAT HAPPENS DURING ACCESS DENIED?

The Buzzer beeps. The photo of the intruder is sent to the owner's email.

WHAT HAPPENS DURING OWNER UPDATE?

When the owner wants to update their face images, three simple but unique security questions are displayed. These questions are already set during the initial setup. The owner must answer all three questions correctly. Only after successful verification, the system allows the owner to update their face images. This prevents any random person from changing the registered owner's image and keeps the smart safe secure.

8. Face Registration

Face registration is the process of capturing the owner’s face and saving it as the authorized identity. This registered face is later used for comparison during recognition.

9. Face Recognition Process

During recognition, the camera captures a new face image. The system checks whether the face matches the registered owner. If the match is successful, it moves to expression verification. If not, the safe remains locked.

10. Access Granted Condition

Access granted means the scanned face belongs to the registered owner. Then a random expression is generated which needs to be performed accurately to be unlocked. LCD displays "Access Granted".

11. Access Denied Condition

Access denied means the scanned face does not match the owner. The system can show “Access Denied”, activate the buzzer, and send the intruder's photo to the owner's email while keeping the safe locked.

12. Role of LCD Display

The LCD display shows live status messages such as scanning, access granted, access denied, next challenge, or safe opened. It helps the user understand what the system is doing.

13. Role of Buzzer

The buzzer works as an alert system. If an unauthorized person tries to access the safe, the buzzer turns on to warn the owner or nearby people.

15. Role of Flask Web App

Flask is used to create the web interface. Through the webpage, the user can register a face, start recognition, update the owner, and view project information.
